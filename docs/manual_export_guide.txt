GUÍA DEL OPERADOR — Exportación manual desde SAP B1
====================================================

Este procedimiento lo hace un humano en la sesión remota de SAP.
Toma ~20-30 min la primera vez, ~10 min en extracciones semanales.

REQUISITOS DEL USUARIO
----------------------
Usuario SAP: FI002 (o cualquier otro con permiso a Query Manager).
Autorización necesaria en SAP:
  • Tools → Queries → Query Manager        (ejecutar)
  • Tools → Queries → Query Generator      (crear)
Si FI002 no tiene Query Manager habilitado, pedir a un keyuser que
active la autorización o usar otro usuario.

MATERIAL

o acceder al repositorio desde la máquina remota.

   Por cada archivo .sql en sql_queries/, en este orden:

     b) Pegar en el editor de Query Manager (reemplazando lo anterior).
     c) Click en "Ejecutar" (botón del reloj) o F5.
     d) Esperar a que termine (abajo dice "N rows returned").

5. EXPORTAR A EXCEL
   a) Con el resultado visible:
        Archivo → Exportar → Microsoft Excel
      o el botón "Exportar a Microsoft Excel" del toolbar.
   b) Guardar con nombre exacto:

      Ejemplo si hoy es 17 de abril 2026:

   c) Guardar en la carpeta compartida del proyecto (Google Drive/

ERRORES FRECUENTES
------------------

"No tiene autorización para ver la tabla X"
   → FI002 no tiene permiso sobre esa tabla. Registrar en
     logs/permissions_missing.txt y pedir al keyuser.

"La consulta tarda demasiado"
   → Cambiar el filtro @SINCE al inicio del .sql a una fecha más reciente,
     por ejemplo @SINCE DATE = '2023-01-01'.

"El Excel exportado abre corrupto"
   → Algunos caracteres raros rompen el export. Intentar exportar a CSV
     en vez de Excel. El pipeline acepta ambos.

"No aparece el menú Query Manager"
   → Puede estar deshabilitado para FI002. Alternativa:
     en la pantalla correspondiente (Listado de artículos, Listado de BPs,
     etc.) usar filtros y el botón "Exportar a Excel" de la pantalla.
